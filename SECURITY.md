# Security Policy

## Reporting a Vulnerability

The security of Flux VPN users is our top priority. If you discover a security vulnerability, we ask that you report it responsibly.

**Please do NOT open a public GitHub issue for security vulnerabilities.**

### How to Report

Send a detailed report to **security@fluxvpn.org** with:

- A description of the vulnerability
- Steps to reproduce the issue
- The potential impact
- Any suggested fixes (optional but appreciated)

### What to Expect

- **Acknowledgment** within 48 hours of your report
- **Status update** within 7 days
- **Fix timeline** communicated once the issue is triaged
- **Credit** in the release notes (unless you prefer anonymity)

## Supported Versions

| Version | Supported |
|:--------|:----------|
| Latest release | ✅ |
| Previous release | ✅ (security fixes only) |
| Older versions | ❌ |

## Security Practices

- All releases are cryptographically signed
- All merge commits to `main` are PGP-signed
- External security audits are conducted regularly
- Audit results are published unredacted in the `audits/` directory
